Large Scale Sass

Jackie Balzer - Behance

Kyle Fiedler - Thoughtbot

Scott Kellum - Vox

Keep Code Simple

  • Other developers should easily understand it.
  • Behance doesn't use a grid framework.
  • Write the css that you need, then refactor it.
  • Scott - Using a hybrid of BEM and SMACSS to keep cascade shallow.

Organization

  • Thoughtbot - uses modified atomic design. Slowly build up class names. Directories follow that structure.
  • Vox - very module focused. With Ruby partials with various components. Easy to find code. Uses variables and nesting to create BEM classes.
  • Behance - uses modules for code org and reuse.

Precautions to not be too clever with naming conventions.

  • Thoughtbot - does pull requests, and guides with standards.
  • Behance - does pull requests, doesn't use BEM, just checks for sanity. Uses scss lint on code. Alphabatizes declarations
  • Vox - works in small teams per site. Each person on team is responsible for meeting standards.

Describe Workflow

  • Behance - has teamlets, comprised of designer, css, js, backend, dev ops, and community teams. Gets code and tasks going quickly. Things go directly to master.
  • Thoughtbot - One designer, two devs.
  • Vox - Small teams typically make for large production.

Is there an effort to bring up skill level after handoff?

  • Thoughtbot - best thing is to pair.
  • Behance - Doesn't have system in place for that.

Has Sass enabled better communication between front-end, back-end, and design?

  • Vox - our small teams typically collaborate well together. All about communication
  • Thoughtbot - Doesn't need that communication bridge. Suggests naming things well.
  • Behance - Has designers that don't code. Developers that don't design. Sass hasn't made Backend (PHP) care about the front-end. But they love how quickly front-end workflows are with Sass. And they like that Sass makes front-end devs more interested in programming.

How much do you use linting?

  • Vox - just had first css lint experience. Turned it off after a week.
  • Thoughtbot - Not much experience
  • Behance - Fucking love Linting! Uses css-lint. Has test suite that runs after pull-request. If it doesn't pass, the pull request doesn't go through

Tips on Performance?

  • Behance - avoiding using lots of tools. Checking performance in Devtools. Run a test. Find what is slow. Clean it up.
  • Thoughtbot - we avoid using selectors that don't perform well. Minification, etc.
  • Vox - Frag Rate checker. Has automated suite test that graphs how sites load over time. Look for spikes and figure out what we commited during that time.

Most memorable deployment fail

  • Vox - issues with fonts!
  • Behance - about to announce stuff in tech crunch, getting lot of exposure, moments before the site was going live, wanted to add blog feed. Wrote PHP for that. Article goes live, and the Blog failed, so then Behance itself failed.

Tips for optimizing

  • Behance - Deleting box shadows
  • Thoughtbot - Yes, and all of css3

Opinions on inlining critical CSS in head of main page?

  • Thoughtbot - Hasn't tried that
  • Behance - Hasn't tried that, uses minification for css and require.js for JS
  • Vox - We don't do that. But there is a tool called Jacket that could help with that.

Average Build Process?

  • Vox - Rails Asset Pipeline and Grunt for SVG stuff
  • Thoughtbot - Rails Asset Pipeline
  • Behance - Custom PHP framework, doesn't have Asset Pipeline. Sass actually isn't compiled in build script.

Sass Organization

  • Behance - lots of partials and reusable modules
  • Thoughtbot - partials
  • Vox - partials, consistent style guide, consistent methodologies

If site styles itself, what are you doing?

  • Behance - CSS team has been doing more JS and PHP

Pattern for theming?

  • Behance - Has multiples galleries with different themes. Use variables once, then update variables for each new theme.

Styling Base Elements?

  • Behance - I do it.
  • Thoughtbot - Bitters
  • Vox - Resets are editable.

Code Conventions? Alphabetical?

  • Behance - Agrees with default scss.lint setup.
